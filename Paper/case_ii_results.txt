Evaluation date: 2026-03-15

Corpus: 451 extracted tables from 54 PDFs (harvested.db)
LLM: llama3.2:latest via Ollama (temperature 0.0, local deployment)
Embeddings: qwen3-embedding:latest via Ollama (B1 only)
Framework: Layer_Experiments_Saab/ (mirrors Case I methodology)

BASELINE DESCRIPTIONS
---------------------
B1: Naive RAG
  - Chunk all extracted_tables into text (heading + table rows)
  - Embed with qwen3-embedding:latest (Ollama, 4096-dim)
  - Retrieve top-5 chunks by cosine similarity
  - Generate answer with llama3.2 in single call
  - No structured schema, no multi-step reasoning

B2: SQL-backed Retrieval
  - Extract product codes from query via regex (RPT/RNT/TFR/C0-series)
  - SQL LIKE search on tablecontent, filename, and table_name fields
  - Pass matching rows directly to LLM for synthesis
  - Single generation step, no validation

B3: RCP (Proposed)
  - 6-stage control loop: GoalDefine -> StrategyPlan -> FunctionExecute ->
    FunctionValidate -> StrategyValidate -> GoalValidate
  - Strategies: SIMPLE LOOKUP / ENHANCED LOOKUP / MULTI-PRODUCT COMPARISON
  - Functions: Extract Keywords -> Normalize Keywords -> Table Search ->
    Filter Table -> Analyze Data
  - Retry with ENHANCED LOOKUP if confidence < 0.6
  - All LLM calls: llama3.2:latest (temperature 0.0)

ERROR ANALYSIS
--------------
Case II errors are concentrated in two categories:

1. Table-navigation failures (Q6, Q8, Q22, Q27-Q28):
   Dimension tables store shell sizes as row indices and parameters as columns.
   The LLM must identify the correct row (e.g., shell size 13) and column
   (e.g., o A max) -- harder than Hydroscand's explicit column-per-attribute schema.
   B3 mitigates this through multi-pass normalization and enhanced filtering.

2. Cross-table ambiguity (Q6 for B2):
   RPT2356 has both a "Dimension table" and a "Panel cutout" table, both containing
   a column labelled "o A". Without strategic disambiguation (only B3 attempts this),
   the LLM retrieves the wrong table's value.

Citation accuracy for B3 (73.3%) exceeds B1 (50.0%) because the RCP's structured
retrieval returns the exact source table with provenance metadata, enabling the LLM
to cite the specific document and section.
